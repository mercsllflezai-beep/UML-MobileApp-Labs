## 🎯 Описание диаграммы классов

Диаграмма классов моделирует систему изучения китайского языка. 

### Основные классы:

**Абстрактный класс `User`** (`id`, `name`, `email`, `passwordHash`) определяет общее поведение:
- `login()` — аутентификация
- `logout()` — завершение сеанса
- `viewProfile()` — просмотр профиля

**От него наследуются:**
- `RegisteredUser` — хранит `progress`, методы: `practiceDialogue()`, `chooseDifficulty()`, `generateNewTask()`, `learnNewWords()`
- `Administrator` — методы: `viewAllUsers()`, `manageUser()`
- `Guest` — метод: `register()` (создаёт `RegisteredUser`)

**Абстрактный класс `Task`** (`id`, `difficulty`, `content`, `answer`, `checkAnswer()`) порождает:
- `DialogueTask` — добавляет `dialogueLines` (диалоговые реплики)
- `WordTask` — содержит `Word` (карточки слов)

**Класс `Word`** хранит:
- `id`, `word` (китайское слово), `translation` (перевод), `examples` (примеры использования)

**Класс `RegisteredUser`** содержит коллекцию `UserProgress` (`completedAt`, `score`), каждый из которых относится к конкретному `Task`.

**Метод `viewProfile()`** возвращает объект `Profile` (`userId`, `name`, `email`, `registrationDate`).

## 📊 Отношения между классами

| Тип отношения | Классы | Описание |
|--------------|--------|----------|
| **Наследование** | `User` ← `RegisteredUser`, `Administrator`, `Guest` | Общие атрибуты и методы |
| **Наследование** | `Task` ← `DialogueTask`, `WordTask` | Типы заданий |
| **Композиция** | `RegisteredUser` ◆ `UserProgress` | Прогресс принадлежит пользователю |
| **Композиция** | `WordTask` ◆ `Word` | Слово принадлежит заданию |
| **Ассоциация** | `UserProgress` → `Task` | Прогресс относится к заданию |
| **Ассоциация** | `Administrator` → `User` | Администрирование пользователей |
| **Зависимость** | `Guest` → `RegisteredUser` | Гость создаёт зарегистрированного |
| **Зависимость** | `RegisteredUser` → `Task` | Пользователь генерирует задания |
| **Зависимость** | `User` → `Profile` | Пользователь возвращает профиль |
